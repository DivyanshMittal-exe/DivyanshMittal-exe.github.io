Kernel Kustomizer: Changing the Linux Kernel v6.1.6

Context Switch Chronicles - Tracking Task Transitions in the Linux Kernel

The first task involved modifying the Linux kernel to track the number of voluntary and involuntary context switch events experienced by a subset of processes and their threads. This was achieved by creating multiple custom system calls. We created a data structure within the kernel to keep track of the monitored processes using a doubly linked list.

Additionally, we developed a kernel module that generated signals for target processes. The module checked a specific file at regular intervals and sent the appropriate signals to the target processes based on the entries in the file, facilitating inter process communication.

Real-Time Rendezvous - Implementing Scheduling Algorithms in the Linux Kernel

We worked on real-time systems and their scheduling algorithms. This assignment involved implementing two scheduling algorithms, namely the deadline monotonic (DM) and rate monotonic (RM) algorithms, within the Linux kernel. The processes were managed using the cached-red-black trees. We also had to make sure that new processes met their timing constraints, including their deadlines, worst-case execution times, and periods, using different checks like the Lehoczky check and the Liu and Layland checks. Using system calls, processes can register and de-register themselves for this scheduler.

We also implemented the Priority Ceiling Protocol (PCP) for the deadline monotonic scheduling algorithm, considering dummy resources with unique resource IDs for locking. These resources were managed using the radix tree. The PCP is a synchronization protocol that helps avoid priority inversions in systems with shared resources. Each resource is assigned a unique resource ID, and the PCP ensures that a task holding a resource with a higher priority prevents lower-priority tasks from accessing that resource. This prevents priority inversions and ensures that tasks meet their deadlines.

Lively LIFO - Crafting a Character Device Driver for a Last-In, First-Out Device

We focused on device drivers in the form of built-in kernel modules. We designed and implemented a character device driver for a virtual LIFO (last in, first out) device. This device had practically unbounded size, limited to 1 MB for the assignment. We created multiple virtual LIFO devices that shared the same driver. A device could be read-only or write-only. Characters written to the write-only device could only be read from the read-only LIFO device, following the LIFO principle. Reading from an empty LIFO device returned an EOF signal, and processes attempting to read were blocked until characters were available. We implemented system calls specific to file operations, such as open, close, read, write, poll, and IOCTL, within the device driver.

Overall, these assignments provided us with valuable hands-on experience in modifying the Linux kernel, implementing scheduling algorithms for real-time systems, and developing device drivers, and much more.

AI Alchemy

Error Correction using Search-based Techniques

In the first assignment, we tackle the intriguing problem of error correction in digitized text. With the increasing use of OCR (Optical Character Recognition) systems to transform handwritten paper records into digital format, errors in the extracted text are inevitable. Our objective is to design a search-based algorithm that can identify and rectify these errors, ensuring the accuracy of the digitized information. To achieve this, we employ a cost function based on language fluency, evaluating the likelihood of different corrections. By exploring the vast space of possible corrections and considering character replacement possibilities using a given error matrix, our algorithm aims to find the most probable correction for a given input sentence.

Connect-4 Game Playing AI Agent

In the second assignment, we embark on an adventure into the realm of game playing AI agents. We focus on developing a formidable agent capable of playing a strategic two-player gameâ€”specifically, an extended version of the popular game Connect4. While the original Connect4 involves players placing tokens vertically in a grid, our extended version introduces additional moves called "PopOut Moves," allowing players to remove bottom-most tokens from specific columns. Our AI agent's mission is to plan its moves within a specified time limit, carefully considering the possible moves and their consequences. We begin by implementing an expectimax agent, pitting it against a random agent, and then refine its abilities using techniques like minimax search and alpha-beta pruning to challenge and defeat an adversarial agent.

Probabilistic Reasoning for Vehicle Localization

In the third assignment, we venture into the realm of probabilistic reasoning, focusing on the captivating domain of vehicle localization. Imagine an autonomous car equipped with a noisy sensor that measures the relative distance between the autonomous vehicle and other cars in its vicinity. Our goal is to develop an algorithm that can estimate a probability distribution (belief) over the possible locations of other vehicles based on the sensor data collected over time. To achieve this, we harness the power of probabilistic trackers, employing a particle filter approach to estimate the current position of each car. Furthermore, we create a planner for the autonomous car, enabling it to navigate safely by utilizing the estimated belief to avoid collisions with other cars and obstacles.

These assignments provide an immersive learning experience, allowing you to delve into the multifaceted world of AI. By working on these projects, you will gain practical knowledge and hands-on experience in developing intelligent agents that can solve real-world problems. From search-based techniques for error correction to game-playing strategies and probabilistic reasoning for vehicle localization, these assignments encompass a diverse range of topics that form the foundation of artificial intelligence. Satellite Savvy: Mapping Indian Water Bodies for Societal Welfare
August 2022-December 2022

We leveraged satellite data from sources like Landsat 7 and Sentinel 2 to classify and analyze water bodies in India. Building upon the existing India SAT project led by Prof. Aaditeshwar Seth, we extended the classification to include water bodies and their seasonal patterns.

Using a random forest classifier, we developed a robust model that accurately distinguishes water bodies from other land types, along with identifying the exact months for which water is present in these bodies. Additionally, we explored alternative models such as support vector machines for comparison.

The information we derived is to be used for government initiatives and schemes related to water resource management. It will decisions about the construction of wells, canals, and other infrastructure, optimizing resource utilization and addressing regional needs.
